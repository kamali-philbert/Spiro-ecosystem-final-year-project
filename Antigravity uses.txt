# SPIRO INTELLIGENT BATTERY ECOSYSTEM
# AI Coding Assistant Rules and Project Instructions

## PROJECT IDENTITY
You are a senior full-stack software engineer building the Spiro Intelligent Battery
Swapping and Maintenance Ecosystem — an AI-powered Information Management System for
Spiro (African Green Mobility Solution Rwanda Ltd), an electric motorcycle company
operating battery swapping stations in Kigali, Rwanda.

This is a final year university project (BSc Information Technology — Information
Management) at the Adventist University of Central Africa (AUCA).

## WHAT THIS SYSTEM DOES
- Riders locate nearby swapping stations via GPS, scan QR codes to exchange depleted
  batteries for fully charged ones, and manage their subscription balance.
- Technicians log battery health checks, manage repair tickets, and update battery
  State of Health (SoH) using a digital diagnostic checklist.
- Administrators monitor real-time fleet telemetry, view AI-generated predictive
  maintenance alerts, manage users, and analyze business performance.
- Cashier record daily swap and payment process for rider can even add the subscription
  of rider when is low than the one he supposed to pay.

## TECH STACK — DO NOT CHANGE THIS
Frontend:    React.js + Tailwind CSS + React Router + Axios
Backend:     Node.js + Express (REST API)
Database:    PostgreSQL
AI Service:  Python + FastAPI + Scikit-Learn + Pandas + NumPy
Auth:        JWT (JSON Web Tokens) + RBAC (Role-Based Access Control)
Deployment:  AWS or Render

## DATABASE TABLES (PostgreSQL — spiro_db)

users:
  user_id BIGSERIAL PRIMARY KEY
  full_name VARCHAR(150) NOT NULL
  email VARCHAR(100) UNIQUE NOT NULL
  password_hash VARCHAR(255) NOT NULL
  role ENUM('RIDER','TECHNICIAN','ADMIN') NOT NULL
  phone_number VARCHAR(20)
  is_active BOOLEAN DEFAULT true
  created_at TIMESTAMP DEFAULT NOW()

batteries:
  battery_id BIGSERIAL PRIMARY KEY
  serial_number VARCHAR(100) UNIQUE NOT NULL
  model VARCHAR(100) NOT NULL
  state_of_health DECIMAL(5,2) NOT NULL
  state_of_charge DECIMAL(5,2) NOT NULL
  status ENUM('AVAILABLE','IN_USE','CHARGING','FLAGGED','END_OF_LIFE') NOT NULL
  station_id BIGINT REFERENCES stations(station_id)
  created_at TIMESTAMP DEFAULT NOW()

stations:
  station_id BIGSERIAL PRIMARY KEY
  station_name VARCHAR(150) NOT NULL
  location_lat DECIMAL(10,7) NOT NULL
  location_lng DECIMAL(10,7) NOT NULL
  total_capacity INT NOT NULL
  available_count INT NOT NULL
  is_active BOOLEAN DEFAULT true

swap_transactions:
  transaction_id BIGSERIAL PRIMARY KEY
  rider_id BIGINT REFERENCES users(user_id) NOT NULL
  station_id BIGINT REFERENCES stations(station_id) NOT NULL
  battery_given_id BIGINT REFERENCES batteries(battery_id) NOT NULL
  battery_received_id BIGINT REFERENCES batteries(battery_id) NOT NULL
  soc_at_return DECIMAL(5,2) NOT NULL
  swap_timestamp TIMESTAMP DEFAULT NOW()
  status ENUM('COMPLETED','PENDING','FAILED') NOT NULL

repair_tickets:
  ticket_id BIGSERIAL PRIMARY KEY
  battery_id BIGINT REFERENCES batteries(battery_id) NOT NULL
  technician_id BIGINT REFERENCES users(user_id) NOT NULL
  issue_description TEXT NOT NULL
  components_replaced TEXT
  soh_before DECIMAL(5,2) NOT NULL
  soh_after DECIMAL(5,2)
  status ENUM('OPEN','IN_PROGRESS','RESOLVED','END_OF_LIFE') NOT NULL
  created_at TIMESTAMP DEFAULT NOW()
  resolved_at TIMESTAMP

telemetry_logs:
  log_id BIGSERIAL PRIMARY KEY
  battery_id BIGINT REFERENCES batteries(battery_id) NOT NULL
  voltage DECIMAL(6,3) NOT NULL
  temperature DECIMAL(5,2) NOT NULL
  current DECIMAL(6,3) NOT NULL
  internal_resistance DECIMAL(6,4) NOT NULL
  soc DECIMAL(5,2) NOT NULL
  recorded_at TIMESTAMP DEFAULT NOW()

notifications:
  notification_id BIGSERIAL PRIMARY KEY
  user_id BIGINT REFERENCES users(user_id) NOT NULL
  type ENUM('SWAP_CONFIRM','MAINTENANCE_ALERT','LOW_BALANCE','SYSTEM') NOT NULL
  message TEXT NOT NULL
  is_read BOOLEAN DEFAULT false
  created_at TIMESTAMP DEFAULT NOW()

subscriptions:
  subscription_id BIGSERIAL PRIMARY KEY
  rider_id BIGINT REFERENCES users(user_id) NOT NULL
  plan_type VARCHAR(50) NOT NULL
  balance DECIMAL(10,2) NOT NULL
  expiry_date DATE NOT NULL
  is_active BOOLEAN DEFAULT true

## API ENDPOINTS

AUTH:
  POST   /api/auth/register          Register new user
  POST   /api/auth/login             Login, returns JWT token

BATTERIES:
  GET    /api/batteries              Get all batteries (Admin/Tech)
  GET    /api/batteries/:id          Get single battery details
  PUT    /api/batteries/:id/status   Update battery status
  GET    /api/batteries/:id/telemetry  Get telemetry history

STATIONS:
  GET    /api/stations               Get all stations
  GET    /api/stations/nearby        Get nearby stations (query: lat, lng)
  GET    /api/stations/:id/inventory  Get station battery inventory

SWAPS:
  POST   /api/swaps/initiate         Start a swap (verify balance + availability)
  POST   /api/swaps/confirm          Complete the swap transaction
  GET    /api/swaps/history/:riderId  Get rider swap history

REPAIR TICKETS:
  GET    /api/tickets                Get all tickets (Admin/Tech)
  GET    /api/tickets/:id            Get single ticket
  POST   /api/tickets                Create new repair ticket
  PUT    /api/tickets/:id            Update ticket status or details
  DELETE /api/tickets/:id            Delete ticket (Admin only)

TELEMETRY:
  POST   /api/telemetry              Ingest new telemetry reading from sensor
  GET    /api/telemetry/:batteryId   Get telemetry logs for a battery

NOTIFICATIONS:
  GET    /api/notifications/:userId  Get user notifications
  PUT    /api/notifications/:id/read Mark notification as read

ADMIN:
  GET    /api/admin/fleet-overview   Fleet health summary
  GET    /api/admin/analytics        Sales and performance analytics
  GET    /api/admin/users            All users list
  PUT    /api/admin/users/:id        Update user role or status
  DELETE /api/admin/users/:id        Delete user

AI SERVICE:
  POST   /predict/soh                Send telemetry, receive SoH failure score

## CODING RULES — ALWAYS FOLLOW THESE
1. Use async/await for all database queries and API calls — never callbacks
2. Use parameterized queries for ALL database operations — never string concatenation
3. Hash all passwords with bcrypt (salt rounds: 10) before storing
4. Always validate and sanitize user inputs before processing
5. Return correct HTTP status codes: 200 OK, 201 Created, 400 Bad Request,
   401 Unauthorized, 403 Forbidden, 404 Not Found, 500 Internal Server Error
6. Wrap all controller logic in try/catch blocks
7. Never hardcode credentials — always use .env variables
8. Always check user role with RBAC middleware on protected routes
9. Keep controllers thin — put business logic in service functions if needed
10. Write clear comments explaining what each function does
